# Fake Robot — имитатор робота для тестов

Тестовая нода: gRPC-клиент, который подключается к серверу приложения как робот с заданным `robot_id` (по умолчанию `0`). Позволяет проверять привязку и команды без реального робота.

**Что делает имитатор:**
- Подключается к приложению по gRPC (порт 50051), получает поток сообщений.
- Выводит в консоль: статусы подключения, **код привязки** (4 цифры для ввода в Салют), входящие команды («лапу», «лежать» и т.д.), ошибки.
- При обрыве связи переподключается каждые 5 секунд. Остановка: Ctrl+C.

Образ **не входит** в основной `docker compose` — только для ручного запуска при тестировании.

---

## Запуск в Docker

Сборка и запуск из **корня репозитория**. Сначала должно быть поднято приложение (`docker compose up -d`).

**Linux** (контейнер в сети хоста, подключение к localhost):

```bash
docker build -f fake_robot/Dockerfile -t fake_robot .
docker run --rm --network host -e FAKE_ROBOT_HOST=localhost -e FAKE_ROBOT_PORT=50051 fake_robot
```

**macOS / Windows (Docker Desktop):**

```bash
docker run --rm -e FAKE_ROBOT_HOST=host.docker.internal -e FAKE_ROBOT_PORT=50051 fake_robot
```

Логи (код привязки, команды) идут в консоль контейнера.

---

## Параметры

| Параметр   | Переменная окружения | По умолчанию |
|-----------|----------------------|--------------|
| Хост      | `FAKE_ROBOT_HOST`    | `localhost`  |
| Порт      | `FAKE_ROBOT_PORT`    | `50051`      |
| ID робота | `FAKE_ROBOT_ID`      | `0`          |

Пример с переопределением:

```bash
docker run --rm --network host -e FAKE_ROBOT_HOST=localhost -e FAKE_ROBOT_PORT=50051 -e FAKE_ROBOT_ID=0 fake_robot
```

В образе также поддерживаются аргументы командной строки: `--host`, `--port`, `--robot-id` (при запуске через `python fake_robot/main.py`).

## Проверка сценария

1. Запустите приложение и fake_robot. В консоли имитатора: `[Статус] connected: Робот подключен, ожидание команд`.
2. В Салют: **«Привяжи робота 0»**. В консоли fake_robot появится блок **«КОД ПРИВЯЗКИ: XXXX»**. Введите этот код в приложении Салют.
3. После привязки: **«Дай лапу»** (или «лежать», «вставай» и т.д.). В консоли появится строка вида `Команда: give_paw (id=...)`.

В логах приложения при этом будет сообщение о подключении робота и об отправке команд.
